Chaos Documentation

This is a stack based language and everything here is determined by stack operations only. As all of you would know the act of placing a value on the stack is called pushing, and the act of taking a value off the stack is called popping.
You will be playing with several stack operations while coding in this language.

The main features of the language are:

  1. Self-Modifying: new instructions can be written into the playfield which can eventually alter the very purpose of the program
  2. Multi-Dimensional: four different contexts can be there to execute the same instruction (series of instructions can be left to right or right to left, or upward or downward). So its not just standard right bottom approach to programming. You have all the four directions available for controlling data flow.

The Programming Aspect: The commands are

Cmd Description
0 – 9 Push this number on the stack
+ add: Pop a and b, then push a+b
- subtract: Pop a and b, then push b-a
* multiply: Pop a and b, then push a*b
/ integer division: Pop a and b, then push b/a, rounded towards 0.
% modulus: Pop a and b from the stack, then push the remainder of the integer division(b/a).
! logical NOT: Pop a value. If value = 0, push 1; else, push 0.
) greater than: Pop a and b, then push 1 if b>a, otherwise zero.
> Start moving right
< Start moving left
^ Start moving up
v Start moving down
o Start moving in a random cardinal direction
_ IF: Pop a value; move right if value=0, left otherwise
| Yet another IF: Pop a value; move down if value=0, up otherwise
[ Duplicate value on top of the stack
" Start stringmode (push ASCII values all the way up to the next ')
= Swap two values on top of the stack
$ Pop value from the stack and discard it
. Pop value and output as an integer followed by a space
, Pop value and output as ASCII character
~ Bridge: Skip next cell
r A way to retrieve data from the storage. Pop y and x, then push ASCII value of the character at that position in the program.
p A way to store a value for later use. Pop y, x, and v, then change the character at (x,y) in the program to the character with ASCII value v.
# Ask user for a number and push it
& Ask user for a character and push its ASCII value
@ End program

Warning

If the stack is empty during popping, this will not generate an underflow! It will simply return '0' to you. Hope you can live with it :)

Also, googling codes isn't gonna work out as the the language has been heavily tweaked out.

Example:

Software Corner!
Code:

25*"!renroC erawtfoS" >[~,_@

Explanation:

25* pushes 10 to the stack,' starts string mode, and second ' sets string mode off.

Second part is the loop that prints values pushed to the stack, > makes PC move right. [ duplicates the current character. Within one iteration the order of events is:

  • duplicate the stack’s topmost element.
  • Check whether it’s equal to 0; if it is, break the loop. During this check the topmost element is always deleted
  • display the topmost element of the stack.

The Program Counter in Detail

The following example will result in a never ending loop :
><

If the PC encounters the 'edge' of the program, for e.g. if you were to try to execute:
< The PC will 'wrap around' to the other 'edge' of the program. This, also, is a never ending loop.

Decision making

The standard 'if' statement here is either '_' or '|', depending on how you want to branch. In both cases a value pops off the stack and it is checked to see if it is true (non-zero,) and change the PCs direction accordingly. '_' behaves as '<' if it's true, and '>' when false. similarly '|'. 'While' loops can be made by putting an 'if' in an infinite loop.

Input

The '#' command will be getting a numeric value from the standard input and the value will be pushed on the stack. '&' will get the next ASCII character from standard input and push it on the stack.

Eg

#, prints out "A" if the user types "65", and...
&. prints out "65" if the user types "A".

Output

The '.' command will pop a value off the stack and display it as an integer, which will be followed by a space. (somewhat like Forth.) ',' will pop a value and display as ASCII but without any space.

Example:

665+*1-,...displays ASCII 65 ("A".), and... 665+*1-....display "65 ".

Special Commands

'~' command skips the next command (which would normally be executed)

For example,
>123...@would lead to the display of "3 2 1 " but
>123~...@would lead to the display of "3 2 " with one of the '.''s being skipped.

')' (back-quote) is the 'greater' command. It performs the comparison of top two values on the stack, and returns '1' if the first is greater than the second.

Example:

65)....leads to the display of "1 " and...
25)....leads to the display of "0 ".
